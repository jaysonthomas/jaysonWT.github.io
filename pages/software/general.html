Mathjax

  • Matrix notation
    • $\begin{bmatrix}a & b\\c & d\end{bmatrix}$
    Rows are terminated with \\\ and columns are separated by &.

  • Display lines
    • The selected answer works. If the equations need not be numbered use \begin{gather*} and \end{gather*}. Use align for alignment.

  • Left alignment of equations
    • Ref. Use & to specify the points in lines that we want aligned with each other.

  • Good link for quick reference
  • Numbering equations
    • \begin{equation}\bh(\bq) = 0.\end{equation}

  • Aligning parts of the equation and adding newlines
    • \begin{equation}\displaylines{ \min_v \quad & \|\bJ(q)v - V_d\|^2_2\\ s.t. \quad & |v|\leq v_{max} }\end{equation} \quad adds 4 spaces.
    & aligns the centres of the left sides (to the ampersand symbol) between themselves. Does the same to the right sides as well.
    \displaylines takes into account \\ to mean 'leave a new line'.

  • 'Sum of' with the limits
    • Good link.

  • Spaces in equations
    • Ref.

  • Underbrace
    • $$u = \underbrace{a_b}_{\text{Test1}} - \underbrace{b_a}_{\text{Test2}}$$ \underbrace{The part of the equation to be labelled }_{\text{The label}}

  • Multiple lines in underbrace
    • Ref. \underbrace{}_{\text{} \atop \text{}} works well.

Json

This shows the general format. I used it as: 

  [
    {"id":"44", "name":"Test"},
    {"id":"55", "name":"Test1"}
  ]
To convert this into a dataframe: 

  with open(inputFile, "r") as reader:
    df = pd.DataFrame(json.loads(reader))
Use json.load() to read a json file.

Git

  • Submodules
    • Ref.

  • Unstage added files
    • 
git reset HEAD
  • Git branch
    • 
# List all branches
git branch -a

# Create branch Y from another, say X
# Changes from branch X will show up as changes from the master on the 
# new branch Y.
git checkout [X]
git checkout -b [Y]

# To merge Z into the newly created branch Y
git merge [Z]

# What branch am I on?
git branch --show

  • Git stash
    • 
# List all the stashes related to different branches with stash indices
git stash list

# Delete a stash
git stash drop index

  • Error when doing a git pull on the new branch created from another (not master)
    • Great explanation - fetch included. 
    
There is no tracking information for the current branch.
Please specify which branch you want to merge with.
See git-pull(1) for details.

    git pull  

If you wish to set tracking information for this branch you can do so with:

    git branch --set-upstream-to=origin/ 7374-adaptive-control
