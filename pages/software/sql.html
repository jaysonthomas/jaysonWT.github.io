Lead


LEAD(timestamp, 1) OVER (PARTITION BY id ORDER BY timestamp ASC) AS nextTs
The above query partitions the huge table by id. Each partition is then sorted by the timestamp column in an ascending order. A new column is then added to the table called nextTs where each entry in it is the next entry of the timestamp column.

This is mainly used in the SELECT clause of a query.
