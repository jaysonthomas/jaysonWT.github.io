Smart pointers

Overview

When a smart pointer is no longer needed (which is the case as soon as it goes out of scope), the memory to which it points is automatically deallocated. In essence, smart pointers are classes that are wrapped around raw pointers. By overloading the -> and * operators, it is possible to use smart pointers with the same syntax as raw pointers. As soon as a smart pointer goes out of scope, its destructor is called and the block of memory to which the internal raw pointer refers is properly deallocated. The technique of wrapping a management class around a resource is called RAII.
