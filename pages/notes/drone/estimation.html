Introduction

Definition

Estimation:
  • Is a temporal process - so you integrate things over time.
  • It's a process of managing uncertainty. We can never be 100% certain where the robot is.

The need for estimation

To quantify uncertainty. The goal is to make a very tight distribution, so that a robot is more certain of its state. The real world (bot and environment) is uncertain in the following ways:
  • It is stochastic. The effect of actions can have uncertain outcomes due to intrinsic or extrinsic factors.
    • We might intend for a vehicle to follow a certain path, but it might get blown to one side by a gust of wind or suddenly stop working. This change in state is a source of uncertainty, we never know when it's going to happen.

  • It is only partially observable. There is uncertainty about the current state of the world.
    • Where is the vehicle? What's behind that building over there? We can't know everything about the world.

How to create an estimator

We use Bayes rule which is a method:
  • To integrate data over time
  • To take multiple sources of information, and come up with a distribution that is the best reflection of what the state might be
Bayes rule relies on probability at its core to quantify uncertainty. There are other ways, such as confidence scores, fuzzy logic etc.

Estimating parameters from data

We do parameter estimation when coming up with a model for a sensor. For example, if we mount a laser rangefinder exactly one meter from a wall, and use it to measure the distance, we probably won't get a measured distance of exactly one meter. It will likely be slightly above, or slightly below.

But if we repeat this measurement many times, we'll likely find that on average the sensor does measure the correct distance plus or minus some error. That error is also called sensor noise, and it's often best modeled as a Gaussian random variable with some mean ideally zero, and some standard deviation.

In order to use that sensor most effectively, we need an accurate estimate of what the parameters of that distribution are. We can't know the exact values of the parameters of a distribution given some data, but we can compute estimates of the parameters in a variety of ways.

For example, a Gaussian is parameterized by $\mu$ and $\sigma^2$. We'll write the estimates of these parameters as $\hat{x}$, and $\hat\sigma^2$ respectively. Now, let's imagine that we have some Gaussian that's generating samples from $x$, with probability density $p(x)$, and we don't know $\mu$, or $\sigma$.

If we repeatedly draw from this distribution to collect samples of data, we'll likely get more data near the mean, and fewer samples out on the tails of the gaussian. Using a technique called the method of moments, we can show that if we have samples $\{x_1, x_2,.., x_n\}$, then $$\hat{x} = \frac{1}{n} \sum^n_{i=1}x_i$$ $$\hat{\sigma}^2 = \frac{1}{n} \sum_{i=1}^n (x_i-\hat{x})^2$$

It's quite convenient that the mean of the distribution matches the expected value, and the variance of the distribution matches that variance.
