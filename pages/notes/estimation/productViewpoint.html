Estimation: Product viewpoint

Introduction

Definition

The goal of estimation is to come up with an estimate $\hat{x}$ for some unknown state $x$, both of which are vectors. Estimation:
  • Is a temporal process - things need to be integrated over time.
  • It's a process of managing uncertainty. It is not 100% certain to know where an object is for example.

The need for estimation

To quantify uncertainty. The goal is to make a very tight distribution, so that a robot is more certain of its state. The real world (bot and environment) is uncertain in the following ways:
  • It is stochastic. The effect of actions can have uncertain outcomes due to intrinsic or extrinsic factors.
    • We might intend for a vehicle to follow a certain path, but it might get blown to one side by a gust of wind or suddenly stop working. This change in state is a source of uncertainty, we never know when it's going to happen.

  • It is only partially observable. There is uncertainty about the current state of the world.
    • Where is the vehicle? What's behind that building over there? We can't know everything about the world.

What does a distribution have to do with robotics?

Solving estimation problems often boils down to using data from sensors which are imperfect.

For example, a range finder pointed at a wall 1m away will most likely give measurements that are slightly different than 1 meter like 0.992, 1.001... etc. But if this measurement is repeated many times, on average, the sensor will yield the correct distance plus or minus some error. This error is called sensor noise and it's often best modeled as a Gaussian random variable with some mean ideally $0$, and some standard deviation.

Therefore, in order to use a sensor more effectively, an accurate estimate of what the parameters of its output distribution are, i.e. a sensor model is required. It is not possible to know the exact values of the parameters of a distribution given some data, but an estimate can be computed.

A good sensor will have a small variance. A bad sensor will have a large variance. Before we put a sensor on a robot we need to estimate what this variance is.
