Estimation: Product viewpoint

Introduction

Definition

The goal of estimation is to come up with an estimate $\hat{x}$ for some unknown state $x$, both of which are vectors. Estimation:
  • Is a temporal process - things need to be integrated over time.
  • Is a process of managing uncertainty. It is not 100% certain to know where an object is for example.

The need for estimation

To quantify uncertainty. The goal is to make a very tight distribution, so that a robot is more certain of its state. The real world (bot and environment) is uncertain in the following ways:

Stochastic

The effect of actions can have uncertain outcomes due to intrinsic or extrinsic factors.

A vehicle intending to follow a certain path might get blown to one side by a gust of wind or suddenly stop working. It is not possible to know when an uncertain event could occur that could then cause a state change.

Partially observable

The real worl is only partially observable. There is uncertainty about the current state of the world.

Where is the vehicle? What's behind a particular object? It is hard to know everything about the world.
